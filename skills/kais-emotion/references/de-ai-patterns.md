# 去 AI 味规则引擎 v2

> 去AI味是为了让情感更真实，不是为了消灭情感。

---

## 一、三级规则系统

### 🔴 P0 禁止级（零容忍）

| ID | Pattern | max_count | Penalty | Fix 建议 |
|----|---------|-----------|---------|---------|
| F001 | `——`（破折号） | 2 | -10/处超出 | 替换为逗号、冒号或括号 |
| F002 | `总而言之\|综上所述` | 0 | -8 | 删掉，直接说结论 |
| F003 | `客观来说\|客观地讲\|理性来看\|从客观角度来看` | 0 | -8 | 删掉，几乎每次都是废话 |
| F004 | `毋庸置疑\|众所周知\|不言而喻\|毋庸赘述\|毋庸讳言` | 0 | -8 | 删掉或给具体来源 |
| F005 | `不可否认` | 0 | -8 | 删掉 |
| F006 | `此外\|然而\|值得注意的是\|更重要的是` | 各1 | -5/处超出 | 用口语替代或直接说 |
| F007 | `让我们\|在这个.*时代\|深刻地\|意义深远\|不可或缺\|至关重要` | 各1 | -5/处超出 | 删掉或换成具体描述 |
| F008 | `不是X，不是Y，而是Z`（否定式排比） | 1 | -5/处超出 | 拆成独立句子 |
| F009 | `！！！\|连续超过3个emoji` | 0 | -10 | 1个感叹号就够 |
| F010 | `作为一个AI\|作为语言模型\|我只是` | 0 | -10 | 删掉，不要提醒用户你是AI |
| F011 | `全面\|革命性\|颠覆\|前所未有\|划时代\|无可比拟\|史无前例` | 各1 | -5/处超出 | 换成具体描述或删掉 |

### 🟡 P1 限制级（控制使用）

| ID | Pattern | max_count | Penalty | Fix 建议 |
|----|---------|-----------|---------|---------|
| L001 | `首先.*其次.*最后`（三段式） | 1 | -3/处超出 | 变化结构：数字列表、直接说、跳过过渡 |
| L002 | `随着.*的发展\|在.*的背景下`（万能开头） | 1 | -3/处超出 | 直接切入主题 |
| L003 | 每段结尾都是总结句 | 30%段落 | -2 | 让观点自然结束，不需要"所以…" |
| L004 | `事实上\|老实说`（假坦诚） | 各1 | -3/处超出 | 删掉或换成"其实" |
| L005 | `诚然\|显而易见\|不容忽视` | 各1 | -3/处超出 | "诚然"→"虽然"；"显而易见"→删掉 |
| L006 | 被动语态（被…/被认为是…） | 20%句数 | -2 | 改为主动：有人发现…、很多人认为… |
| L007 | 翻译腔（"它是…的"结构、"这导致…的发生"） | 各1 | -3/处超出 | "它是…的"→直接陈述；"导致…发生"→"所以…" |

### 🟢 P2 建议级（改善质量）

| ID | Check | Penalty | 理由 |
|----|-------|---------|------|
| S001 | 有第一人称"我" | -3 | 没有"我"的文章读起来像说明书 |
| S002 | 句子长度有变化 | -2 | 全长句或全短句都像机器 |
| S003 | 有口语化表达 | -2 | 适当穿插口语，不完全书面 |
| S004 | 有具体数字/细节 | -2 | "显著提升"不如"快了2秒" |
| S005 | 允许不完美 | -1 | 偶尔重复、稍微啰嗦，反而有人味 |

## 二、情感与去AI味协同

### 核心原则

**情感优先级高于风格规则。** 去AI味不该让文字变得冷冰冰。

### 冲突处理表

| 冲突场景 | 处理方式 |
|---------|---------|
| 规则要求删"！但场景需要热情 | 保留1个，删除多余的 |
| 口语化替换后情感变淡 | 用语气词补偿："不错啊"、"可以啊" |
| 删总结句后结构散了 | 用"所以"或"核心就是"轻收 |
| 用户很悲伤但规则要求简洁 | 情感优先，允许稍长，但不给鸡汤 |
| P0词恰好是最佳表达 | 允许1次，标注为有意为之 |

### 场景特殊规则

| 场景 | 放宽的规则 | 收紧的规则 |
|------|-----------|-----------|
| 安慰 | 允许更多口语、语气词、重复表达 | 禁止鸡汤、禁止"你会好的" |
| 鼓励 | 允许感叹号（≤2）、短句密集 | 禁止空洞赞美、"你一定可以" |
| 批评 | 允许更直接、更犀利 | 禁止"客观来说"、禁止委婉兜圈 |
| 辩论 | 允许反问、短句连续 | 禁止"不可否认"、禁止假客观 |
| 故事 | 允许情绪化用词、感叹、重复 | 禁止总结句、禁止"这个故事告诉我们" |
| 教学 | 允许"这个有点绕"等口语 | 禁止"值得注意的是"、禁止三段式 |
| 日常 | 全面放宽P2 | 只守P0底线 |

## 四、增强版检测清单

### 基础检查（必做）

```
1. 搜索 "——" → 超过 2 处？
2. 搜索 "此外" → 超过 1 处？
3. 搜索 "然而" → 超过 1 处？
4. 搜索 "值得注意的是" → 出现了？
5. 搜索 "总而言之" → 出现了？
6. 搜索 "综上所述" → 出现了？
7. 搜索 "客观" → 出现了？
8. 搜索 "不是…而是" → 否定式排比超过 1 处？
9. 搜索 "让我们" → 出现了？
10. 搜索 "！！！或连续emoji" → 出现了？
11. 统计句子长度 → 全是中等长度？
12. 有没有第一人称"我"？
```

### 情感检查（有情感注入时做）

```
13. 情感匹配场景吗？（安慰别用鼓励语气）
14. 有没有假共情？（"我理解你的感受"但没有具体内容）
15. 感叹号数量合适吗？（日常≤1，鼓励≤2）
16. 有没有鸡汤/空洞赞美？
17. 情感浓度是否均匀？（全篇一样浓 = 假）
```

### 场景检查（按场景做）

```
18. 安慰：有没有说"你会好的"？（不该说）
19. 鼓励：有没有说"你一定可以"？（太空洞）
20. 批评：有没有用"有待改进"？（太委婉）
21. 辩论：有没有用"不可否认"？（假客观）
22. 故事：有没有总结句？（不该有）
23. 教学：有没有"值得注意的是"？（该删）
```

### 评分标准

| 分数 | 等级 | 处理 |
|------|------|------|
| ≥ 80 | ✅ 通过 | 直接输出 |
| 60-79 | ⚠️ 需修改 | 按检测清单修复后重新评分 |
| < 60 | ❌ 重写 | 回退改造，最多2轮 |
