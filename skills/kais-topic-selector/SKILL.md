# Kais Topic Selector — 通用AI内容选题技能

目标：构建一套**平台 × 形式 × 风格 × 题材**多维度的通用AI内容选题系统，从热点发现到脚本大纲一站式输出。不限于漫剧，覆盖所有AI辅助内容创作形式。

## 适用范围

- AI漫剧 / AI漫画 / AI短视频 / AI图文 / AI播客 / AI仿真人剧
- 多平台适配（抖音 / 小红书 / B站 / 快手 / 微博 / 视频号 / YouTube / TikTok）
- 多风格覆盖（搞笑 / 悬疑 / 情感 / 知识科普 / 恐怖 / 职场 / 玄幻 / 日常 / 商业）
- 选题库管理、竞争分析、爆款拆解、变现路径规划

## 🎯 系列 Profile（Series Profile）

为连载/系列内容定义**固定 DNA**，后续所有选题自动受 Profile 约束，确保系列一致性和方向稳定性。

### Profile 是什么

Profile 是一个系列的"宪法"——定义了不可妥协的核心属性。选题器在为该系列生成选题时，所有候选选题必须通过 Profile 的四重过滤。

### Profile 结构

```yaml
# 存储位置: data/kais-topics/profiles/{series_id}.yaml
series_id: dream-poetry          # 系列唯一标识
series_name: 梦境词话            # 系列名称
version: 1.0

# === 不可妥协的约束 ===
constraints:
  # 1. 核心定位（这个系列是什么）
  positioning: "用AIGC让诗意变成肉眼可见的画面"
  positioning_anti: "不是讲诗，是让你看见诗"

  # 2. 情绪目标（观众看完应该有什么感受）
  emotion_target: "太美了"
  emotion_anti: "不明觉厉、看不懂、无聊"

  # 3. 受众策略（为谁做）
  audience_strategy: "每个人都能共鸣"
  audience_anti: "文化精英、行业内部、小众圈层"

  # 4. 视觉方向（画面风格约束）
  visual_direction: "现实场景与梦幻场景的无缝融合"
  visual_anti: "纯写实、纯幻想、割裂感"

  # 5. 变现目标
  monetization_goal: "涨粉变现"
  monetization_anti: "纯艺术表达、无商业化考量"

  # 6. 内容形式
  format: "AI短视频 55-60秒 竖屏 9:16"
  platform_primary: "B站"
  platform_secondary: "小红书（切片）"

# === 涨粉四大原因（选题评分加权项）===
growth_factors:
  - factor: "受众基数决定天花板"
    weight: 25%
    description: "选题必须不挑人，无文化门槛，所有人都能代入"
    check: "我爸妈能看懂并被打动吗？"
  - factor: "情绪匹配最精准"
    weight: 20%
    description: "治愈+共鸣是最强情绪驱动，选题必须触发这个情绪"
    check: "看完有没有'这不就是我吗'的感觉？"
  - factor: "视觉差异化是杀手锏"
    weight: 20%
    description: "现实+梦幻融合市场上几乎没有竞品，必须强化这个优势"
    check: "这个画面是即梦能做但别人没做过的吗？"
  - factor: "系列化能力强"
    weight: 15%
    description: "素材取之不尽，不会枯竭"
    check: "这个方向能做20期不重复吗？"

# === 选题红线（绝对不做）===
red_lines:
  - "需要专业背景知识才能看懂"
  - "纯抽象无具体场景"
  - "视觉风格与其他AI视频雷同"
  - "说教或给结论"
  - "超过60秒"

# === 首期方向锁定 ===
first_episode:
  direction: "日常诗意"     # B 日常诗意
  topic: "半梦半醒的那2秒"
  style: "现实+梦幻融合"
  status: "🟡 策划中"
```

### Profile 在选题流程中的作用

```
热点扫描（100+话题）
  ↓
  ↓ ← 🔒 Profile 红线过滤（排除不符合的）
  ↓
选题漏斗（30→15→5→2-3）
  ↓
  ↓ ← 🔒 Profile growth_factors 加权评分（涨粉潜力排序）
  ↓
最终选题输出（自动标注 Profile 匹配度）
```

### 使用方式

**1. 创建 Profile**（系列启动时一次性定义）
```
用户: 为梦境词话创建系列profile
→ 引导用户填写 positioning / emotion_target / audience_strategy / visual_direction
→ 存储到 data/kais-topics/profiles/dream-poetry.yaml
```

**2. 加载 Profile**（后续每次选题自动加载）
```
用户: 梦境词话下一期选题
→ 自动加载 profiles/dream-poetry.yaml
→ 所有候选选题按 Profile 约束过滤和评分
→ 输出时标注每个选题的 Profile 匹配度（高/中/低）
```

**3. 更新 Profile**（系列迭代时调整）
```
用户: 梦境词话 profile 加一条红线：不要深夜场景
→ 更新 profiles/dream-poetry.yaml
→ 后续选题自动遵守
```

### Profile 评分公式

```
选题总分 = 基础分(100分) × Profile加权系数

Profile加权系数 = Σ(factor_weight × factor_match_score)

factor_match_score:
  - 完美匹配: 1.0
  - 基本匹配: 0.7
  - 部分匹配: 0.4
  - 不匹配: 0.0
  - 触碰红线: -0.5（直接淘汰）
```

## 🔄 选题流程

### Step 1：需求确认

```yaml
platform: 小红书      # 目标平台
format: AI图文漫      # 内容形式
style: 搞笑          # 风格
topic: null          # 题材方向（可选）
count: 5             # 生成选题数量
audience: null       # 目标受众（可选）
series: false        # 是否连载
tone: null           # 基调（正能量/暗黑/中立）
reference: null      # 参考账号/作品（可选）
market: domestic     # domestic / overseas
```

如果用户没有指定，**直接用默认值开始，不要逐个问**。

### Step 2：热点扫描

**执行动作：**
1. 使用 `web_search` 搜索当前热点（3次搜索）
2. 搜索关键词模板：
   - `{platform} {format} 爆款 {month} {year}`
   - `{topic} 热点 梗 {year}`
   - `{platform} 涨粉 内容趋势 {year}`
3. 如有参考账号，搜索其近期作品和数据分析
4. **政策扫描**（必做）：搜索`{format} 新规 政策 监管 {year}`，评估合规风险
5. **IP热点**（推荐）：搜索`热门网文IP 漫剧改编 {year}`，评估IP改编机会
6. **日历节点扫描**（推荐）：搜索`{month} 节日 热点 营销节点 {year}`，提前2-3周布局节日/电商节点选题
7. 如有参考账号，搜索其近期作品和数据分析

### Step 3：选题漏斗生成

```
广度扫描（100+热点话题）
  ↓ 合规筛选（排除政策风险/备案不可行内容）
  ↓ 热度筛选（前30个，基于搜索热度+趋势）
  ↓ 平台适配（前15个，匹配平台格式/受众/算法/账号等级）
  ↓ 差异化评估（前5个，与现有内容对比）
  ↓ 可行性验证（2-3个，AI工具链+制作难度+投流ROI）
  ↓ 最终选题输出
```

**每个选题必须包含：**

```yaml
选题:
  id: 1
  title: "标题"
  hook: "引流钩子（一句话描述为什么值得看）"
  premise: "内容前提（2-3句话）"
  target_audience: "目标受众"
  platform_fit: "平台适配分析"
  format: "推荐内容形式"
  heat_score: 8/10       # 热度预估
  diff_score: 7/10       # 差异化评分
  feasibility: 中         # 制作难度（低/中/高）
  emotion: "核心情绪驱动"
  ai_tools: "推荐AI工具链"
  tags: ["标签1", "标签2"]
  similar_hits: "参考爆款案例"
  monetization: "推荐变现路径"
```

### Step 4：竞争分析（推荐）

对 Top 2 选题执行（对判断市场饱和度至关重要）：
1. 搜索同类内容数量和表现
2. 分析差异化空间
3. 评估市场饱和度
4. 给出竞争策略建议

### Step 5：脚本大纲（用户确认选题后）

```yaml
脚本大纲:
  标题: "xxx"
  格式: "{platform}适配格式"
  总帧数/时长: 8帧 / 60秒
  分镜:
    - 帧1:
        画面描述: "xxx"
        角色: "xxx"
        对白/旁白: "xxx"
        AI绘画提示词: "xxx"
        预估时长: "3秒"
        情绪点: "xxx"
  结尾钩子: "引导关注/评论/追更"
  发布建议:
    标题文案: "xxx"
    话题标签: "#xxx #xxx"
    发布时间: "建议时段"
    封面建议: "xxx"
```


### 情绪驱动模型
| 情绪类型 | 内容策略 | 典型选题 |
|---------|---------|---------|

| 😱 恐惧 | 氛围营造、jump scare | 恐怖怪谈、都市传说 |

### 黄金1.8秒钩子（2026更新）

> ⚠️ 2026年观众决策时间已缩短至1.8秒。悬念式开头完播率反降12%，**价值前置**更有效。前3秒留存权重占整体评估40%。

**有效策略**：具体痛点+解决方案+结果承诺（前5秒内完成）
- **价值前置型**："用这个构图法，照片马上专业感翻倍"（推荐，停留率+23%）
- **反转型**："我以为...没想到..."
- **共鸣型**："这不就是我的生活吗？"
- **冲突型**："当xxx遇到xxx..."

**互动设计**（新流量密钥）：
- 视频中留3个可讨论的点
- 结尾提出选择题（"你觉得A还是B更好？"）
- 当评论数>点赞数1/3时，系统自动触发二次推荐（+37%流量）
- 抖音新增"评论质量分"隐藏指标，直接影响下一轮流量分配

## 💰 变现路径

### 六大变现路径（从易到难）
1. **平台分账**：低粉即可，分成最高90%（红果/快手）
2. **广告变现（IAA）**：免费+广告，用户规模大
3. **付费解锁（IAP）**：初期测试，正在被IAA取代
4. **品牌商单**：高粉账号接高价商单
5. **IP延伸**：系列开发+衍生品
6. **出海分发**：TikTok/YouTube，差异化题材

### IAAP模式（行业主流）
```
IAP测试 → IAA长线运营 → 会员分账 → IP开发 → 出海
```

### 变现决策树
- **有IP基础** → IAA + IP延伸（保底+高分成模式）
- **无IP纯原创** → IAA + 平台分账 + 快速试错
- **有粉丝基础** → 品牌商单 + IP授权
- **出海题材** → TikTok + YouTube + 本地化改编

### ⚠️ 变现风险提示
- **90%创作者在亏损**：头部效应极其明显（来源：钛媒体2026-03）

- **收入参考**：抖音内容付费月入1000-80万元（差异极大）；爆款单部净利润20-30万
- **头部案例**：酱油文化2025年营收10亿（千人团队）；灵矩动漫月产能100部（700人）

### IP改编策略（2026主流）
- 大多数AI漫剧爆款来自网文IP改编，而非纯原创
- 主要版权方：阅文集团、番茄小说、知乎、七猫

## 📈 SEO/算法优化

### 平台算法要点
- **抖音**：完播率（核心权重40%）→ 前3秒留存 → 互动率 → 分享率 → 搜索关键词匹配
- **小红书**：浏览量 → 收藏+互动 → 标签匹配 → 账号权重 → 搜索排名
- **B站**：弹幕率 → 完播率 → 一键三连 → 垂类标签 → 搜索收录
- **视频号**：社交推荐 → 完播率 → 互动 → 朋友点赞加权
- **TikTok**：For You完播率 → 互动率 → Share → Follow → 搜索关键词
- **YouTube**：Watch Time（核心）→ CTR → Engagement → SEO关键词 → 推荐算法

### 关键词策略（四类关键词布局）

| 关键词类型 | 说明 | 示例 | 布局位置 |
|-----------|------|------|---------|
| **核心词** | 行业大词，搜索量大竞争高 | AI漫剧、搞笑漫画 | 标题+话题标签 |
| **长尾词** | 精准定位，竞争低转化高 | 打工人AI搞笑漫剧 | 标题+描述+字幕 |
| **疑问词** | 对应用户搜索意图 | AI怎么做漫剧？ | 标题/开头文案 |
| **场景词** | 情境关联，触发联想 | 加班摸鱼搞笑漫剧 | 标题+描述+标签 |

**关键词挖掘方法**：
- 抖音搜索框下拉联想（输入核心词看推荐）
- 巨量算数/百度指数 查看搜索趋势
- 竞品爆款标题提取高频词
- 平台热搜榜/话题榜捕捉实时热词

### 标题优化公式

| 平台 | 标题长度 | 推荐公式 | 示例 |
|------|---------|---------|------|
| **抖音** | ≤30字 | `情绪词+数字+悬念+核心关键词` | "99%打工人不知道的3个摸鱼技巧😱" |
| **小红书** | ≤20字 | `痛点+解决方案+emoji` | "加班到崩溃？这5个方法救了我😭" |

| **YouTube** | ≤60字符 | `核心关键词+价值主张` | "How I Made an AI Comic in 30 Days - Full Tutorial" |

**标题通用技巧**：
- 核心关键词尽量靠前（前10个字符权重最高）

## 🔐 合规与备案（独立步骤）

> 2026年4月1日起，AI漫剧正式进入"持证上岗"时代（来源：腾讯新闻2026.03）

### 备案制度要点

**分层审核制度**（2026新规）：
- **投资≥100万**：需直接对接广电部门备案
- **投资<100万**（绝大多数中小创作者）：仅需配合合作平台完成自审并提供材料，由平台统一上报省级广电备案

### 备案操作流程

```
选题阶段 → 内容自查（版权/敏感内容/合规底线）
  ↓ 制作阶段 → 如实核算成本
  ↓ 制作完成 → 通过合作平台提交自审材料
  ↓ 平台审核 → 平台统一上报省级广电
  ↓ 获得备案号 → 正式上线发布
```

**时间预估**：
- 规划备案：**2个工作日**（上海微短剧大会2026标准）
- 上线备案：**5个工作日**
- 选题时需预留 **1-2周备案时间**

### 内容自查清单
- [ ] 版权清晰（非直接复制已有作品设定/情节）
- [ ] 无敏感政治/宗教内容
- [ ] 暴力/恐怖内容符合分级标准
- [ ] 角色形象无侵权风险
- [ ] AI生成内容已标注
- [ ] 成本核算真实可查

### 对选题的影响
- **"蹭热点"玩法已失效**：新规要求预留备案时间，无法快速追热点
- **提前规划**：节日/热点内容需提前2-3周启动，留出备案周期
- **版权合规**：网文IP改编需确认授权链路完整
- **长期内容优先**：连载/系列内容比单期热点内容更适合新规环境

## ⚠️ 注意事项

1. **热点时效性**：选题必须基于最近7天的热点数据
2. **平台差异**：同一选题在不同平台需要调整格式、时长、节奏
3. **AI能力边界**：标注哪些画面AI难以实现（复杂动作、多人互动）
4. **版权意识**：避免直接复制已有作品的设定和情节；65.6%用户对漫剧版权有顾虑
5. **数据驱动**：选题评分基于搜索数据，不是拍脑袋
6. **情绪为王**：爆款的核心是情绪匹配，不是制作精良度
7. **IP壁垒**：大多数爆款来自网文IP改编（阅文/番茄/知乎/七猫），故事是最硬壁垒
8. **免费趋势**：IAA（广告）模式正在取代IAP（付费），优先考虑免费内容
