# Kai's 小红书运营技能 v2

目标:构建可复用的"小红书运营"流程,让任何账号类型都能复用同一套动作框架。

> v2 更新(2026-04-05):基于 AI辅助知识付费小红书成功案例深度研究,重点强化选题策略--从"凭灵感选题"升级为"数据驱动选题体系"。

## 适用范围(默认即通用流程)

- 账号定位与内容方向
- **数据驱动选题**（搜索需求挖掘、爆款筛选、选题裂变、选题库管理）
- 内容生产（AI 辅助 + 人工注入真实体验）
- 竞品/同类账号对标
- 小红书发布前演练与内容交付
- 发布后快速复盘（互动结构、评论回复、热点追踪）
- **变现路径规划**（付费笔记、商单、虚拟产品、知识付费）
- **发布节奏与测试策略**（起号→增长→稳定三阶段）
- Viral Copy 链路（输入 URL，高贴合学习封面/配图、标题、正文并生成可发布近似结构笔记）

将每类账号的行业细节作为"案例模块(case module)"挂载到通用流程中。

## 常用术语

- `选题`:可发布、可讨论、可转发的内容切入点
- `引流钩子`:标题/开头一句用于触发停留与点击
- `结构化输出`:标题、正文、互动问句、话题、标签五元组
- `快照`:用于验证页面状态的关键证据快照
- `回放`:流程失败后重试或改道执行

## 0) 启动与环境校验(所有任务都遵循)

执行前先按 `references/xhs-runtime-rules.md` 中"运行规则"执行,优先遵循失败可复用顺序。

- 固定使用内置浏览器 profile:`openclaw`,出现通道异常先切回后再重试。
- 若 browser(openclaw-manager)能力处于 disabled/不可用:先执行一次轻量重试(如 status/profiles),仍不可用则进入故障引导,明确告知用户"当前浏览器工具未启用",并引导用户按文档启用后再继续(参考:`https://docs.openclaw.ai/tools/browser`)。
- 以 `evaluate` 为先,关键节点少量 `snapshot`,单步动作最多重试一次。
- 失败后保留已获结果,切稳健路径并汇报。

## 1) 技能默认行为(所有任务都遵循)

- **先读本技能目录下的 `persona.md`**(小红书平台专用人设/语气/发布与回复风格)。所有对外文案(发帖/评论回复/私信话术)都必须遵循。

## 2) 账号定位(可复用)

每个账号先确认 4 个变量:

- 目标用户:年龄/场景/痛点(如「下班后碎片时间」「追星讨论人群」)
- 内容价值主张:每篇给用户什么(观点、情绪价值、实操建议)
- 差异化角度:同类账号不做什么、你做什么
- 风格规范:语气、长度、冲突边界(避免过激)

输出:

- 人设关键词(3-5)
- 内容支柱(3 个)
- 口头禅/固定句式(2-3 个)
- 不能碰底线(红线)清单(剧透、人身攻击、虚假承诺)

## 2.1) 账号分析

按 `references/xhs-account-analysis.md` 执行。

- 默认采样最近 9-15 篇内容做轻量体检
- 从定位、内容结构、互动转化、辨识度、可持续性 5 个维度判断
- 输出必须包含"最大优势、最大短板、下一步动作"

## 3) 数据驱动选题体系(v2 核心)

> 核心洞察:小红书已是"第二个百度",用户带着"解决问题"的需求来搜索。选题不是靠灵感,而是靠**搜索需求挖掘**。
>
> 完整 SOP 见 `references/xhs-topic-research-v2.md`。

### 选题三原则

1. **跟着已火的做** — 爆款是平台验证过的,复制验证过的方向
2. **瞄准搜索需求** — 用户搜什么关键词,就做什么选题
3. **聚焦垂直细分** — 不做"美妆"做"学生党平价护肤"

### 选题流程概览

| 步骤 | 动作 | 产出 |
|------|------|------|
| 1. 搜索需求挖掘 | 小红书搜索框下拉词 + 交叉验证 | 长尾关键词清单 |
| 2. 数据化筛选 | 采集高互动笔记,按赞粉比/收藏比/评论率筛选 | 候选选题池(10-20条) |
| 3. 对标博主监控 | 分析 Top 30 内容 + 评论区挖掘 | 选题金矿 + 爆款模板 |
| 4. 量化评分 | 5维度10分制打分 | 优先创作清单 |
| 5. 选题裂变 | 角度裂变×3 + 风格裂变 + 模板沉淀 | 批量选题产出 |
| 6. 选题库沉淀 | 存入知识库,持续积累 | 长期选题资产 |

### 选题清单格式(每条必须包含)

- 选题标题(20字内)
- 观点标签(支持/反对/中性)
- 互动钩子
- 证据来源(数据 + 赞粉比)
- 风险提示
- **数据评分**:热度(0-2) + 匹配(0-2) + 互动(0-2) + 可写(0-2) + 风险(0-2),总分≥7优先

### 选题灵感

按 `references/xhs-topic-ideation.md` 执行,结合 v2 数据驱动方法。
- 默认输出 3-5 条,带互动钩子、三段结构、数据评分
- 优先从选题库中筛选已验证的选题,而非凭空发散

## 4) 通用内容模板(小红书)

每次产出至少 2 个备选:

- 标题(争议/立场/反问,≤20字优先)
- 开头钩子(1-2 句)
- 正文(3 段:观点→证据→反方)
- 互动提问(1 句)
- 话题(5-8 个)
- 风险标注(是否剧透 / 引战边界 / 版权风险)

### v2 内容生产要点(基于实战研究)

**AI 辅助创作的正确姿势:**
1. **不要让 AI "写一篇小红书笔记"** - 找到同领域爆款笔记,提取结构和话术风格,作为 AI 的参考模板
2. **AI 生成框架,人注入灵魂** - AI 产出骨架,但必须加入真实体验、使用细节、对比感受。这是"AI味"→"真人味"的关键
3. **口语化改写** - AI 初稿偏书面语,需要改写成小红书的口语化风格,自带"种草感"和"真实感"
4. **违禁词检测** - 发布前必须过违禁词检测(广告法违禁词、平台敏感词、行业违规词),否则辛苦写的笔记可能被限流

**"真实性护城河"原则:**
- 纯 AI 生成内容是死路一条,平台正在加强打击
- AI 的核心价值 = 提升效率 + 拓展思路,成品必须经过深度人工参与
- "人在环路"(Human-in-the-loop)模式是确保内容质量与账号安全的根本

## 5) 通用发布链路(不发稿)

详细发布执行路径请直接按 `references/xhs-publish-flows.md` 执行,避免重复维护。

发布前必须满足的核心点:

- 账号先登录创作后台,确认页面在 `openclaw` profile 可操作。
- 明确发布类型(视频 / 图文 / 长文),三要素:封面、标题、正文。
- 到达"发布"按钮可见处停手,默认不直接点击发布。
- 若涉及截图确认,优先附件形式发送到飞书,并在用户确认后再发布。

## 8) 变现路径(v2 新增)

> 详细变现 SOP 见 `references/xhs-monetization-v2.md`

### 变现里程碑

| 阶段 | 粉丝门槛 | 变现方式 | 预期收益 |
|------|----------|----------|----------|
| 冷启动 | 0粉 | 专注涨粉,不开变现 | 0 |
| 首次变现 | 100粉 | 付费笔记功能 | 小额 |
| 商单接入 | 1000粉 | 蒲公英平台接商单 | 200-2000元/条 |
| 多元变现 | 1000+粉 | 商单+虚拟产品+知识付费 | 月入1-3万 |
| IP变现 | 1万+粉 | 课程+私域社群+企业定制 | 年入30万+ |

### 虚拟产品(高利润赛道)

净利润率 80%-90%,品类包括:
- Notion/飞书模板(日程规划、项目管理等)
- 资料包(行业报告、学习笔记、Prompt合集)
- 设计素材(封面模板、排版模板)

### 关键原则
- 100粉就开始布局变现,不要等"准备好了"
- 虚拟产品是性价比最高的变现方式(零库存、零物流)
- 知识付费:9.9元引流款 + 999元进阶款

## 9) 发布节奏与测试策略(v2 新增)

基于实战案例数据,推荐以下日更节奏:

| 阶段 | 日产出 | 核心目标 |
|------|--------|----------|
| 起号期(1-2周) | 3-5篇 | 测试选题方向,积累数据 |
| 增长期(3-4周) | 5-8篇 | 找到爆款模型,批量复制 |
| 稳定期(2个月+) | 3-5篇 | 持续优化,专注变现 |

**测试→复制循环:**
1. 日更 3-5 篇,持续 2 周观察数据
2. 找到赞粉比高、互动好的内容方向 = 爆款模型
3. 批量复制爆款模型,日更提升到 5-8 篇
4. 持续迭代,淘汰表现差的方向

**2026 新趋势:**
- 平台明确发力"中长视频",2分钟以上视频完播超30秒可进更大流量池
- 可结合即梦 AI 等工具低成本产出视频内容

## 10) 失败与修复(必须遵循)

- 自动化失败先重试一次(同策略)
- 仍失败则改道:换到"更稳妥同义路径"
- 不做无效重复动作;保留当前进度可复用,报告一次用户需手动的单一动作
- 若知识库暂时不可写,先返回结构化摘要,任务结束后补记,不阻塞主流程

## 实操经验(持续有效)

- **统一规则:所有浏览器操作一律走内置浏览器 profile=`openclaw`**(除非用户明确要求使用 Chrome 扩展 Relay)。
- 文字配图是稳定写入口,typed text 直接成为封面文案
- 发布话题优先用 UI 选题,不建议纯文本粘贴大量 `#话题`
- `evaluate` 批量改写富文本时,尽量少改版式,避免丢失 topic entity
- 关键步骤前保留一次快照,可用于复盘与问题定位
- `发布` 按钮可见 ≠ 发布成功;必须明确标注"到发布页停手"
- 若出现新类型评论节奏问题,优先减少每小时回复密度而非提高频率

### v2 补充经验
- 选题优先从选题库筛选已验证的方向,不要每次从零开始
- AI 写文案必须喂爆款模板,不要让 AI "写一篇小红书笔记"
- 发布前必须过违禁词检测,否则可能被限流
- 爆款模型找到后要批量复制,但保持 20% 差异化避免被判同质化
- 100粉就可以开始变现布局,不要等"准备好了"
