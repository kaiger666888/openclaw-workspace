# GEO (Generative Engine Optimization) 分析 Skill

分析内容被 AI 搜索引擎引用的潜力，输出评分卡和具体优化建议。

## 什么是 GEO

GEO（Generative Engine Optimization）是针对 AI 搜索引擎优化内容可见性的新兴学科。与传统 SEO 追求排名不同，GEO 的目标是让 AI 引擎在回答用户问题时**引用和推荐你的内容**。

**海外引擎**：ChatGPT、Perplexity、Google AI Overviews、Claude、Gemini
**国内引擎**：DeepSeek、Kimi（月之暗面）、豆包（字节跳动）、文心一言（百度）、通义千问（阿里）、秘塔AI搜索、智谱清言

关键数据：AI 引用流量同比增长 527%（2025），SEO 和 AI 结果重叠率仅 10-39%，说明这是完全不同的优化逻辑。

## 参数

| 参数 | 选项 | 默认 | 说明 |
|------|------|------|------|
| `input_type` | url / markdown / text | url | 输入内容类型 |
| `depth` | quick / standard / deep | standard | 分析深度 |
| `engines` | chatgpt, perplexity, google-ai, claude, gemini, deepseek, kimi, doubao, wenxin, tongyi, metaso, zhipu, all-overseas, all-china, all | all | 目标 AI 引擎（支持 all-overseas / all-china 快捷分组） |

### 参数解析规则

- **url**：用户提供 URL → 用 `web_fetch` 抓取内容
- **markdown**：用户提供 markdown 文件路径 → 用 `read` 读取
- **text**：用户直接提供文本 → 直接分析
- 如果用户未指定参数，默认 `url + standard + all`
- 如果用户只给了 URL 没说其他参数，直接用默认值，不要问

## 工作流程（必须严格遵循）

### Step 1: 内容获取

根据 `input_type` 获取待分析内容：

| input_type | 工具 | 说明 |
|------------|------|------|
| url | `web_fetch` | 抓取页面，提取 markdown 内容 |
| markdown | `read` | 读取本地文件 |
| text | 无需工具 | 直接使用用户提供的文本 |

如果 URL 抓取失败，尝试 `web_fetch` 换 `extractMode: "text"` 重试一次。仍失败则报告错误。

### Step 2: 内容分析（6维度逐项评分）

对获取的内容按以下 6 个维度评分（每项 0-100 分）：

#### 维度 1: 可引用性（权重 22%）⚠️ 实证第二强预测因子（r=0.368）

评估内容是否包含 AI 引擎会引用的独特信息。

| 分数区间 | 标准 |
|----------|------|
| 80-100 | 包含原创数据/统计、独特定义、独家案例、原创研究 |
| 60-79 | 有具体数字和案例，但不够独特 |
| 40-59 | 以模糊描述为主，缺少具体数据 |
| 0-39 | 纯观点无支撑，或内容过于通用 |

**检查清单**：
- 是否有具体数字/统计？（不是"很多""一些"这种模糊词）
- 是否有原创定义或独特观点？
- 是否有独家案例或研究？
- 是否有可直接引用的金句/结论？

#### 维度 2: 结构化程度（权重 14.4%）实证 r=0.294

评估内容的组织结构是否便于 AI 解析和提取。

| 分数区间 | 标准 |
|----------|------|
| 80-100 | 清晰 H1-H3 层级 + 表格 + 列表 + FAQ + Schema 标记 |
| 60-79 | 有标题层级和列表，缺表格或 FAQ |
| 40-59 | 有基本分段，但结构混乱 |
| 0-39 | 大段连续文本，无任何结构 |

**检查清单**：
- 是否有清晰的标题层级（H1→H2→H3）？
- 是否使用列表（有序/无序）？
- 是否有表格？
- 是否有 FAQ 格式（问答对）？
- 是否有 Schema.org 结构化数据标记？

#### 维度 3: 权威性（权重 30%）🔴 实证最强预测因子（r=0.454）

评估内容的可信度和 E-E-A-T 信号。**权威性是AI引擎引用决策的第一优先级**（arXiv 2509.08919: AI搜索系统性地偏好Earned Media而非品牌自有内容）。

| 分数区间 | 标准 |
|----------|------|
| 80-100 | 作者署名+资质、引用学术论文/权威研究、机构背书、被其他权威平台引用、有原始数据或实验支撑 |
| 60-79 | 有来源引用但学术引用不足，作者信息基本完整 |
| 40-59 | 有部分引用但来源偏商业/不可靠，缺乏学术支撑 |
| 20-39 | 仅引用商业网站或社交媒体，无学术/官方来源 |
| 0-19 | 无来源引用、无作者信息、纯个人观点 |

**检查清单**：
- 是否有作者署名和专业背景？（Experience）
- 是否引用了学术论文或权威研究？（Expertise + Authoritativeness）
- 是否有原始数据、实验或调查支撑？（Trustworthiness）
- 引用来源是否多样化？（不依赖单一来源）
- 是否被 Wikipedia、学术数据库、行业报告引用？（Earned Media 信号）
- 是否有 E-E-A-T 信号？（经验/专业/权威/可信）

**关键洞察（来自实证研究）**：
- AI搜索引擎对"Earned Media"（第三方权威来源）有系统性偏好（arXiv 2509.08919）
- 在 YMYL 领域（健康、法律、金融），用户先验证可信度再深入阅读（Previsible 2025）
- AI流量集中在"决策页面"（行业页1.14%渗透率、工具页0.95%），而非浏览页面

#### 维度 4: 多平台覆盖（权重 9.6%）实证 r=0.188

评估内容/品牌在 AI 引擎偏好平台上的存在度。

| 分数区间 | 标准 |
|----------|------|
| 80-100 | Reddit + Wikipedia + YouTube + 行业平台均有存在 |
| 60-79 | 在 2-3 个平台上可见 |
| 40-59 | 仅在自有网站存在 |
| 0-39 | 在线存在度极低 |

**检查方法**：用 `web_search` 搜索品牌名/核心主题 + 平台名，检查是否存在。

**关键平台优先级**：
- Reddit（Perplexity 46.7% 引用来自 Reddit）
- Wikipedia（ChatGPT 47.9% 引用来自 Wikipedia）
- 小红书（国内AI引擎高频引用，尤其 Kimi、秘塔）
- 知乎（DeepSeek、豆包、通义千问等偏好）
- 微信公众号（国内AI引擎重要来源）
- B站、CSDN、掘金等技术社区
- YouTube、LinkedIn、G2、行业论坛

#### 维度 5: 内容新鲜度（权重 12%）实证 r=0.275

| 分数区间 | 标准 |
|----------|------|
| 80-100 | 30天内更新，有明确发布/更新日期 |
| 60-79 | 90天内更新 |
| 40-59 | 1年内更新 |
| 0-39 | 超过1年未更新，或无日期信息 |

**检查清单**：
- 是否有明确的发布日期？
- 是否有"最后更新"日期？
- 内容是否反映最新情况？
- 数据和引用是否过时？

#### 维度 6: AI 引擎友好度（权重 12%）实证 r=0.332

| 分数区间 | 标准 |
|----------|------|
| 80-100 | Schema 标记 + FAQ + 结构化数据 + 机器可读格式 |
| 60-79 | 有部分结构化标记 |
| 40-59 | 基本可解析但无结构化增强 |
| 0-39 | 内容难以被机器解析（图片文字、PDF扫描件等） |

**检查清单**：
- 是否有 Schema.org 标记（JSON-LD/Microdata）？
- 是否适合被直接提取为摘要？
- 段落是否独立完整（不依赖上下文）？
- 是否有清晰的问答对格式？

### Step 3: GEO 评分计算

加权总分 = 权威性×0.30 + 可引用性×0.22 + 结构化程度×0.144 + 新鲜度×0.12 + AI引擎友好度×0.12 + 多平台覆盖×0.096

四舍五入到整数。

### Step 4: 优化建议生成

根据各维度得分，从优化策略库（见下方）中选取最相关的建议。规则：
- 每个维度最多选 2 条建议
- 优先选择针对低分维度的建议
- 按预期提升幅度排序
- 每条建议格式：**[问题]** → **[行动]** → **[预期效果]**

### Step 5: 平台特定建议

根据用户指定的 `engines` 参数，给出针对各 AI 引擎的差异化建议（见平台差异指南）。

### Step 6: 输出报告

根据 `depth` 参数按对应格式输出（见下方）。

## 平台差异指南

### ChatGPT
- **偏好来源**：Wikipedia（47.9% 引用）、新闻网站、教育资源
- **引用逻辑**：倾向于引用"公认知识"和权威定义
- **优化重点**：确保 Wikipedia 有收录，内容风格偏向教育性/百科性
- **关键词策略**：使用清晰的定义和概念解释

### Perplexity
- **偏好来源**：Reddit（46.7% 引用）、新鲜内容（90天内）、技术社区
- **引用逻辑**：偏好真实用户讨论和最新信息
- **优化重点**：在 Reddit 等社区建立存在感，保持内容更新频率
- **关键词策略**：包含具体数据和时效性信息

### Google AI Overviews
- **偏好来源**：已有高排名内容、强 E-E-A-T 信号、结构化数据
- **引用逻辑**：与现有搜索排名高度相关，E-E-A-T 信号强的内容优先
- **优化重点**：SEO 基础 + E-E-A-T 强化 + Schema 标记
- **关键词策略**：与传统 SEO 兼容，同时增强结构化

### Claude
- **偏好来源**：公开数据较少，推测偏好学术来源和长篇深度内容
- **引用逻辑**：更审慎，倾向于有充分证据支撑的声明
- **优化重点**：内容深度和论证严密性
- **关键词策略**：提供完整的论证链条而非结论

### Gemini
- **偏好来源**：Google 生态内内容、YouTube、Google Scholar
- **引用逻辑**：与 Google 搜索结果有较高重叠
- **优化重点**：Google 生态布局 + Scholar 收录
- **关键词策略**：确保 Google 索引完整

### DeepSeek（深度求索）
- **偏好来源**：知乎、微信公众号、技术博客、GitHub、学术论文
- **引用逻辑**：技术问题偏好 GitHub 和技术社区，通用问题偏好知乎和公众号
- **优化重点**：在知乎回答问题、公众号产出专业内容、GitHub 开源项目有文档
- **关键词策略**：中文长尾关键词、技术术语、具体代码示例
- **特点**：国内开发者首选，对中文技术内容覆盖极好

### Kimi（月之暗面）
- **偏好来源**：小红书（高权重）、知乎、学术论文、微信公众号、豆瓣
- **引用逻辑**：偏好生活类/消费类内容来自小红书，专业类来自知乎和论文
- **优化重点**：小红书种草内容优化、知乎专业回答布局
- **关键词策略**：场景化长尾词（"适合25岁的..."、"2026年最值得..."）
- **特点**：长文本处理能力强，对深度长文有偏好

### 豆包（字节跳动）
- **偏好来源**：抖音生态内容、今日头条、小红书、百度百科
- **引用逻辑**：与字节跳动内容生态深度绑定，短视频文案和头条文章权重高
- **优化重点**：抖音/头条内容布局、百科词条建设
- **关键词策略**：短视频相关话题词、热门话题标签
- **特点**：用户基数最大（国内），偏生活化和娱乐化

### 文心一言（百度）
- **偏好来源**：百度百科、百度知道、百度文库、百家号
- **引用逻辑**：与百度搜索生态强绑定，百家号内容权重高
- **优化重点**：百度百科词条、百度知道回答、百家号原创
- **关键词策略**：与百度 SEO 兼容，长尾关键词
- **特点**：与百度搜索高度集成，传统 SEO 基础可直接复用

### 通义千问（阿里）
- **偏好来源**：淘宝/天猫商品内容、1688、阿里云文档、微信公众号
- **引用逻辑**：电商类问题偏好阿里系平台，技术问题偏好官方文档和社区
- **优化重点**：淘宝/天猫商品描述优化、阿里云技术文档
- **关键词策略**：电商相关关键词、技术文档关键词
- **特点**：电商场景优势明显

### 秘塔AI搜索
- **偏好来源**：学术论文（高权重）、微信公众号、知乎、行业报告
- **引用逻辑**：最"学术"的国内AI搜索引擎，偏好有出处的信息
- **优化重点**：学术论文引用、行业数据引用、来源标注规范
- **关键词策略**：学术性关键词、研究类查询
- **特点**：无广告、引用溯源清晰，适合专业内容

### 智谱清言
- **偏好来源**：微信公众号、知乎、技术博客、开源社区
- **引用逻辑**：与 GLM 模型训练数据相关，偏好中文互联网主流内容
- **优化重点**：公众号+知乎双平台布局
- **关键词策略**：中文自然语言查询
- **特点**：中规中矩，覆盖面广

## 实现说明

此 Skill 使用 OpenClaw 内置能力：
- `web_fetch` — 抓取 URL 内容

无需额外安装。

**实验方法**：Playwright抓取30个URL → 6维度评分 → Pearson/Spearman相关分析 → 权重迭代校准

**关键指标**：
| 指标 | 结果 |
|------|------|
| Pearson r | 0.416（中等相关） |
| p值 | 0.044（统计显著） |
| Cited vs Uncited 差值 | 17.3分 |
| F1（阈值40分） | 0.82 |

**维度有效性排名**：
1. 权威性 r=0.454 — 域名权威+E-E-A-T信号是最强预测因子
2. 可引用性 r=0.368 — 独特数据/统计/定义显著提升引用率
3. AI引擎友好度 r=0.332 — 结构化数据+Schema标记
4. 结构化程度 r=0.294 — 标题层级+列表+表格
5. 内容新鲜度 r=0.275 — 年份+更新信号
6. 多平台覆盖 r=0.188 — 多平台存在感

**局限性**：样本量30个（偏小），代理指标为搜索排名（非真实AI引用），仅测试英文URL。建议每季度重新验证。
