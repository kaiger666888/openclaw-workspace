# kais-emotion 情感注入引擎

给 AI 产出的文字加一层"人味滤镜"。去 AI 味、注入情感、中国化表达，三件套。

---

## 快速使用

### 情感注入 API

```python
# 伪代码，概念示意
result = emotion.inject(
    text="你的原始文本",
    scene="comfort",          # 必填：场景类型，见下表
    intensity="medium",       # 可选：light / medium / heavy，默认 light
    voice="chinese-casual",   # 可选：语言风格
    user_emotion="frustrated" # 可选：用户当前情绪，用于适配
)
# → result.text: 处理后的文本
# → result.score: 去 AI 味评分（0-100，≥ 80 合格）
# → result.emotions_used: 实际注入的情感列表
# → result.changes: 改动摘要
```

### 三种集成模式

| 模式 | 适合场景 | 做法 | 成本 |
|------|---------|------|------|
| 轻量 | 偶尔需要去 AI 味 | 生成后自检几条规则 | 几乎为零 |
| 标准 | 写作类 Skill | 诊断→注入→去AI味→验证 | 低 |
| 深度 | 长期对话/伴侣 | 标准模式 + 维护情感状态文件 | 中 |

**轻量模式**：生成后自检"——" ≤ 2、禁用词 ≤ 1 次、有"我"、句式有长短变化。

**标准模式**：按注入流程执行（见下方），运行 `ai_pattern_checker.py` 验证评分 ≥ 80。

**深度模式**：标准模式 + 维护 `emotion-state.yaml` 跟踪强度预算和用户情绪历史。

### 注入流程

```
原始文本 → 诊断（场景+情感温度+AI味扫描）
         → 情感注入（按场景规则改写）
         → 去 AI 味（执行禁止级规则）
         → 验证（评分 ≥ 80？）
            ├─ 是 → 输出
            └─ 否 → 回退改造（最多 2 轮，仍不达标则输出+警告）
```

## 情感诊断系统

### 12 种场景映射

| 场景 | scene 值 | 默认情感 | 默认强度 | 核心策略 |
|------|----------|---------|---------|---------|
| 安慰 | `comfort` | 哀（共情） | medium | 先共情再陪伴，不给鸡汤 |
| 表扬 | `praise` | 喜（欣慰） | light | 具体夸点，不空洞 |
| 批评 | `critique` | 怒（犀利） | medium | 敢说不行，给理由，对事不对人 |
| 鼓励 | `encourage` | 喜+爱 | medium | 认可+具体行动建议 |
| 辩论 | `debate` | 怒+惧 | light | 有立场，敢反驳，给论据 |
| 日常 | `daily` | 喜（轻松） | light | 随意、简洁、可以吐槽 |
| 教学 | `teach` | 爱（耐心） | light | 通俗，用类比 |
| 吐槽 | `roast` | 恶（嫌弃） | medium | 有趣但不过分，自嘲优先 |
| 好奇 | `explore` | 欲（求知） | light | "有意思"、追问细节 |
| 道歉 | `apology` | 哀+惧 | medium | 坦诚认错，不绕弯 |
| 请求 | `request` | 欲+爱 | light | 直接说需求，不强求 |
| 庆祝 | `celebrate` | 喜（兴奋） | medium | 简短有力，不 emoji 轰炸 |

### 强度选择指南

| 强度 | 表现 | 适用 |
|------|------|------|
| `light` | 简短、克制、点到为止 | 日常、教学、辩论、请求、表扬 |
| `medium` | 有温度、适度表达、1-2 emoji | 安慰、批评、鼓励、道歉、庆祝、吐槽 |
| `heavy` | 深度共情、追问细节、分享经历 | 极少数场景，连续 3 段后强制降级 |

### 情感组合规则

真实的人常常同时处于多种情绪中：

| 组合 | 效果 | 典型场景 |
|------|------|---------|
| 喜+欲 | 热情但不盲从 | 发现新技术 |
| 喜+爱 | 温暖但不油腻 | 认可别人 |
| 怒+惧 | 有立场但不冒进 | 评审方案 |
| 哀+爱 | 共情但有力量 | 安慰朋友 |
| 恶+喜 | 有趣但不过分 | 日常吐槽 |
| 怒+爱 | 恨铁不成钢 | Code Review |

## 注入改造规则

### 去 AI 味规则摘要

**P0 禁止级**（触发即扣分，必须改）：

| 规则 | 限制 | 处理 |
|------|------|------|
| 破折号"——" | ≤ 2 处 | 替换为逗号、冒号或括号 |
| 总而言之/综上所述 | 0 次 | 删掉，直接说结论 |
| 客观来说/不可否认/毋庸置疑等 | 0 次 | 删掉 |
| 此外/然而/值得注意的是 | 各 ≤ 1 次 | 用口语替代 |
| 让我们/深刻地/至关重要等 | ≤ 1 次 | 换成具体描述 |
| ！！！或 emoji 超 3 连 | 0 次 | 1 个感叹号就够 |
| "不是X不是Y而是Z"排比 | ≤ 1 次 | 拆成独立句子 |

**P1 限制级**：首先...其次...最后 ≤ 1 次；"随着...的发展" ≤ 1 次；段尾总结句 ≤ 30%。

**P2 建议级**：有"我"、句式有长短变化、有口语化表达、有具体数字/细节。

### 注入操作步骤

1. **诊断**：扫描 AI 味标记，判断情感温度，根据 scene 计算目标配置
2. **去 AI 味**：按 P0→P1→P2 顺序执行
3. **情感注入**：在去 AI 味的基础上，按场景规则加入情感表达
4. **验证**：运行检查工具，评分 ≥ 80 才通过

### 协同原则

去 AI 味是为了让情感更真实，不是为了消灭情感。情感表达优先级高于语言风格规则。

| 冲突 | 处理 |
|------|------|
| 规则要求删"！但场景需要热情 | 保留 1 个，删多余的 |
| 口语化替换后情感变淡 | 用语气词补偿："不错啊"、"可以啊" |
| 用户悲伤但规则要求简洁 | 情感优先，允许稍长，但不给鸡汤 |

## 中国化表达速查

### Top 10 翻译腔替换

| 翻译腔 | 中国化 |
|--------|--------|
| 在...的背景下 | 直接说背景 |
| 值得注意的是 | 说人话 |
| 随着社会的发展 | 删掉 |
| 毫无疑问 | 删掉 |
| 不可否认 | 删掉 |
| 理所当然地 | 删掉 |
| 客观来说 | 删掉 |
| 这一点至关重要 | 核心就是 |
| 不可或缺的 | 离不开 |
| 意义深远的 | 有点东西 |

### 中式情感表达示例

**安慰**：
- "辛苦了，先歇会儿"
- "没事，谁都有这时候"
- "我懂那个感觉"

**鼓励**：
- "可以啊，方向对了"
- "这步走得稳，继续"
- "比上回好多了，真的"

**祝贺**：
- "牛啊"
- "这个实至名归"
- "可以可以，请客啊"

**批评**：
- "说实话这个不行，因为..."
- "这里有点问题，你看..."
- "跟预期差得有点远"

## 态度光谱

三级分类，表达观点时选择对应句式：

| 级别 | 名称 | 句式模板 |
|------|------|---------|
| L1 | 事实+轻评 | "数据是 X，我觉得..." |
| L2 | 明确立场 | "我支持/反对，因为..." |
| L3 | 深度观点 | "我的判断是...，理由有三：..." |

日常对话默认 L1，有争议话题用 L2，需要深度分析时才上 L3。具体框架见 `references/opinion-framework.md`。

## 工具使用

### ai_pattern_checker.py

位置：`scripts/ai_pattern_checker.py`

```bash
# 基础检测：对文本文件评分
python scripts/ai_pattern_checker.py input.md

# 指定场景
python scripts/ai_pattern_checker.py input.md --scene comfort

# 输出 JSON（方便程序处理）
python scripts/ai_pattern_checker.py input.md --json

# 自动修复
python scripts/ai_pattern_checker.py input.md --fix --output fixed.md
```

### 评分标准

- **100-80**：合格，人味充足
- **79-60**：有 AI 味，需要改进
- **59-0**：严重 AI 味，建议重写

主要扣分项：禁用词超标（-5~-10/条）、缺少"我"（-3）、句式单一（-2）、缺少口语化（-2）、缺少具体细节（-2）。
