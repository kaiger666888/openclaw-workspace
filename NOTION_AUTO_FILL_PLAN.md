# Notion自动填充内容解决方案

## 问题分析

当前的 `daily-tasks.sh` 脚本只创建了空白模板，没有填充实际内容。

## 解决方案

### 方案一：使用子代理生成内容（推荐）

使用 `sessions_spawn` 在后台运行子代理，让AI根据当前日期和事件类型生成真实内容。

**优点**：
- 内容质量高，AI可以根据实际情况生成
- 可以调用web_search获取最新信息
- 完全自动化

**缺点**：
- 需要消耗AI tokens
- 生成时间较长（需要等待子代理完成）

**实现方式**：

在cron任务中，设置 `payload.kind = "agentTurn"`，让子代理：
1. 搜索相关内容（如GitHub Trending、科技新闻）
2. 总结成markdown格式
3. 调用notion-cli更新页面

### 方案二：提供初始示例内容

为每个任务类型提供一些初始的、结构化的示例内容。

**优点**：
- 立即可用
- 无额外成本
- 用户提供明确的填充方向

**缺点**：
- 内容不是实时的
- 需要手动更新或手动填充

### 方案三：混合方案（推荐用于生产环境）

结合两种方案：
1. 创建页面时，提供结构化的模板和示例
2. 同时触发子代理在后台生成内容
3. 子代理生成完成后，自动更新页面

## 推荐实施步骤

### 第一步：更新cron配置

将当前的cron任务从简单的systemEvent改为agentTurn：

```json
{
  "name": "daily-news-with-content",
  "schedule": {
    "kind": "cron",
    "expr": "0 7 * * *",
    "tz": "Asia/Shanghai"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "请执行每日新闻任务：1) 使用web_search搜索今日科技新闻；2) 总结成markdown格式；3) 使用notion-cli更新到每日新闻页面（$DATE）",
    "model": "default",
    "timeoutSeconds": 300
  },
  "sessionTarget": "isolated",
  "enabled": true
}
```

### 第二步：创建内容生成技能

创建一个专门的内容生成skill，包含：
- web_search调用
- 内容格式化
- notion页面更新逻辑

### 第三步：渐进式迁移

1. 先保留现有模板创建逻辑
2. 添加子代理内容生成作为可选增强
3. 验证稳定后，将主要逻辑迁移到子代理

## 需要用户确认的选择

请选择您希望的方案：

1. **完全自动化**：使用子代理生成真实内容（需要消耗tokens，较慢但质量高）
2. **模板化**：提供更好的示例模板（快速，但内容固定）
3. **混合方案**：先创建模板，后台异步生成内容
4. **手动填充**：保持现状，只创建空白框架供用户手动填写
