# XHS 数据驱动选题 SOP（v2）

> 基于小红书 AI 成功案例深度研究（2026-04）提炼的选题方法论
> 核心转变：从"凭灵感选题"到"数据驱动选题"

## 0. 核心认知

- **小红书 = 第二个百度**：用户带着"解决问题"的需求来搜索
- **搜索词 = 用户真实需求**：搜索框里频繁出现的短句，就是最好的选题
- **爆款可复制**：爆款一定是在平台上已经验证过的，新手不要"创新"，要"复制已验证"
- **评论区 = 选题金矿**：用户提问、抱怨、纠结就是下一个爆款选题

## 1. 选题三原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 跟着已火的做 | 爆款是平台验证过的，复制验证过的方向 | 看到"平价护肤"火了，做"学生党平价护肤" |
| 瞄准搜索需求 | 用户搜什么，做什么选题 | 搜索框下拉词 = 选题方向 |
| 聚焦垂直细分 | 不做大而全，做细分领域 | 不做"穿搭"，做"小个子通勤穿搭" |

## 2. 选题信号采集三步法

### Step 1：搜索需求挖掘

**执行方式：用浏览器（profile=openclaw）打开小红书搜索**

1. 输入核心关键词（如"减肥""护肤""副业"）
2. 截取搜索框下拉联想词（通常 8-10 个）
3. 截取"大家都在搜"推荐词
4. 记录每个词的搜索热度排序

**交叉验证来源：**
- 抖音搜索框下拉词
- 5118 关键词工具
- 百度指数下拉词

**产出：长尾关键词清单**
```
核心词：减肥
长尾词：减肥最快方法、学生党减肥、三伏天减肥、产后减肥食谱、减肥不反弹...
```

### Step 2：数据化选题筛选

**执行方式：用每个关键词在小红书搜索，采集前 10 条笔记数据**

采集字段：
- 标题
- 点赞数、收藏数、评论数
- **赞粉比** = 点赞/粉丝数（低粉高赞 = 可复制性强）
- **收藏比** = 收藏/点赞（收藏>点赞 = 实用型内容，更易被推荐）
- 发布时间
- 内容形式（图文/视频）

**筛选标准：**
- 赞粉比 > 10（如果博主粉丝 < 1000，点赞 > 10000）
- 收藏比 > 1（收藏数 > 点赞数）
- 评论数 > 50（有互动潜力）
- 发布时间 < 30天（时效性）

**产出：候选选题池（10-20 条）**

### Step 3：对标博主监控

**一次性分析（新发现博主时）：**
1. 打开对标博主主页
2. 采集最近 30 条内容的标题、互动数据
3. 找出赞粉比 Top 5 的内容，分析其选题规律
4. **重点看评论区**：用户在问什么、抱怨什么、纠结什么
5. 评论区的高频问题 = 下一个爆款选题

**持续监控（已关注博主）：**
- 定期检查新发内容
- 第一时间跟进热点选题
- 提取爆款结构和话术风格，存为模板

## 3. 量化评分体系

每条候选选题按以下维度打分（每项 0-2 分，总分 10 分）：

| 维度 | 0分 | 1分 | 2分 |
|------|-----|-----|-----|
| 热度信号 | 无相关爆款 | 有类似内容但表现一般 | 多个爆款验证过 |
| 账号匹配 | 完全不相关 | 部分相关 | 高度匹配人设和内容支柱 |
| 互动潜力 | 预计评论<10 | 预计评论10-50 | 有争议/可站队，预计评论>50 |
| 可写性 | 缺乏素材和信息 | 有部分素材 | 有充足案例和数据 |
| 风险分 | 无风险 | 轻微擦边 | 高风险内容 |

**筛选规则：**
- 总分 ≥ 7：优先创作
- 总分 5-6：备选
- 总分 < 5：淘汰
- 风险分 = 2：直接淘汰（无论总分）

## 4. 爆款选题裂变

找到一篇爆款后，执行以下裂变策略：

### 4.1 角度裂变（同一选题 × 3 角度）
- 立场型：支持/反对/中立但有条件
- 过程型：怎么做/怎么避坑/怎么判断
- 结果型：做完会怎样/为什么别人做不到

### 4.2 风格裂变（同一选题 × 多风格）
- 不同账号使用不同语气和风格
- 同一内容可以改编成图文、视频、长文多种形式

### 4.3 模板沉淀
从爆款中提取可复用元素：
- 标题句式模板
- 封面信息层级
- 正文节奏结构
- 互动机制设计
- 话题标签组合

## 5. 选题库管理

**推荐工具：飞书多维表格**

字段设计：
| 字段 | 说明 |
|------|------|
| 主题 | 选题核心词 |
| AI发散选题 | AI 基于主题生成的多个角度 |
| 主题分类 | 按内容支柱分类 |
| 选题来源 | 搜索词/对标博主/评论区/热点 |
| 参考链接 | 原始爆款笔记链接 |
| 赞粉比 | 原始数据 |
| 评论率 | 原始数据 |
| 量化评分 | 10分制 |
| 状态 | 待写/已写/已发布/已复盘 |
| 数据表现 | 发布后的实际互动数据 |

**日常习惯：**
- 刷小红书时看到好选题，随手扔进选题库
- AI 一句话就能自动搭建表格和分类
- 日积月累，选题库就是"流量预测引擎"

## 6. 失败与降级

1. 搜索信号不足 → 降级到"账号定位 + 常见标题骨架"生成基础选题
2. 对标博主数据难采集 → 降级到首页推荐流分析
3. 量化评分数据缺失 → 降级到人工判断，但标记"待验证"
4. 禁止在信号不足时硬编"爆点"
