# 每日任务质量改进计划

## 问题诊断

### 1. 搜索 API 限流
- **现象**: Brave Search API 频繁限流
- **影响**: 内容质量下降，被迫使用"已知知识"
- **解决**: 
  - 增加 fallback 搜索源（DuckDuckGo、Google）
  - 缓存搜索结果，减少重复请求
  - 分散搜索时间，避免集中调用

### 2. 内容深度不足
- **现象**: 新闻条数少、技术研究缺少代码
- **影响**: 信息价值低
- **解决**:
  - 每日新闻：至少 10 条，按重要性排序
  - 技术研究：必须包含 1-2 个代码示例
  - AIGC 总结：必须包含数据对比表格

### 3. 格式应用不完整
- **现象**: 颜色、高亮块使用少
- **影响**: 视觉效果单调
- **解决**:
  - 更新 notion-block-templates.json，增加颜色样式示例
  - 关键信息用红色/蓝色标注
  - 重要提示用 callout + 颜色

### 4. 超时问题
- **现象**: 3 分钟超时导致任务失败
- **影响**: 需要手动重试
- **解决**:
  - 将超时时间从 180s 增加到 300s
  - 减少单次搜索量，优先完成核心内容
  - 如果搜索失败，直接用已知知识补充

## 格式改进规范

### 颜色使用规则
- 🔴 **红色**: 警告、失败、负面信息
- 🔵 **蓝色**: 技术术语、链接提示
- 🟢 **绿色**: 成功、正面信息、增长数据
- 🟡 **黄色背景**: 需要注意的信息
- 🟣 **紫色**: 投资相关、金融术语

### Callout 使用规则
- 💡 **提示**: 使用建议、技巧
- ⚠️ **警告**: 风险提示、注意事项
- ✅ **成功**: 完成状态、正面结果
- 📊 **数据**: 关键数字、统计信息

### 结构规范
```
H1 标题
├── Callout（核心摘要）
├── H2 分区
│   ├── 段落
│   ├── 列表
│   └── 代码块（如适用）
├── H2 分区
│   └── ...
└── 来源/参考
```
