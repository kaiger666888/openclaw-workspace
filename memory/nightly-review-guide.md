# 夜间回顾任务指南

## 任务目标
每晚回顾当天所有对话，提取重要信息并更新 MEMORY.md

## 执行时间
每天凌晨 01:00 (Asia/Shanghai)

## 回顾清单

### 1. 扫描今日对话
- 查看当天的 memory/YYYY-MM-DD.md
- 回顾 sessions_history 获取完整对话上下文

### 2. 提取信息类型

#### 🎯 优先级更新
- 新的紧急任务 → P0
- 进行中项目进展 → P1
- 新项目意向 → P2

#### 📝 决策记录
- 重要决策及其背景原因
- 改变主意的事情及原因

#### 📚 知识捕获
- 新学到的概念/技巧
- 有价值的资源链接
- 值得记住的事实

#### 🔗 关系信息
- 提到的人名、项目名
- 重要的 ID（Notion 页面、数据库等）

#### 📊 项目进展
- 完成的任务
- 遇到的障碍
- 下一步计划

### 3. 更新 MEMORY.md

#### 优先级区域
```
## 🎯 当前优先级
- 更新 P0/P1/P2
- 移动完成项到 Archives
```

#### PARA 结构
```
## 📁 PARA 结构
- Projects: 更新项目状态
- Areas: 记录领域相关进展
- Resources: 添加新资源
- Archives: 归档已完成项目
```

#### 决策记录
```
## 📝 关键决策记录
- 添加今日决策
```

### 4. 渐进式总结原则
- **Layer 1**: 原始笔记保留在 daily memory
- **Layer 2**: 重要内容提炼到 MEMORY.md
- **Layer 3**: 真正关键的提炼为"优先级"区域
- **Layer 4**: 核心洞察进入 AGENTS.md 或 USER.md

### 5. 输出格式
完成后在当前会话发送简短总结：
```
🌙 夜间回顾完成

✅ 更新内容：
- [列出主要更新]

📊 今日统计：
- 对话轮次：X
- 新增决策：X
- 项目进展：X
```

## 注意事项
- 保持简洁，避免信息过载
- 关注"可操作"的信息
- 尊重隐私，敏感信息不入库
- 渐进式总结，层层提炼
