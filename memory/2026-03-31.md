1. ❌ mmdc (mermaid-cli) + `--no-sandbox` → Puppeteer沙箱限制

6. **核心发现**: Mermaid handDrawn是商业版(Mermaid Chart)功能，开源版不支持
7. ✅ **最终方案**: Node.js + rough.js 直接生成手绘SVG → resvg-js 转PNG

**技术栈**:
- `roughjs` (npm) - 手绘风格生成器（generator模式，非SVG DOM模式）
- `@resvg/resvg-js` (npm) - SVG→PNG 高质量渲染
- 输出: `/tmp/xhs-uml-images/class-handdrawn.png`, `seq-handdrawn.png`, `fix-handdrawn.png`

3. **Diff 类图优化**（commit `0f276c7`）：隐藏未变更类、限制方法数量、颜色标注变更类型

#### 序列图核心问题
- **初始序列图完全不可用**：200+ 行，把 for/if 内部迭代和构造函数都当成了跨类调用
- **用户需求**：展示 commit 间变更的 UML 图，变更代码不限深度，未变更代码限制深度
- **多轮修复**：
  - `calm-ridge`：变更代码不限深度 → 测试失败
  - `clear-ke`：只追踪跨类调用，过滤 self-call → 大幅改善（~20行 vs 200+行）
  - `brisk-cr`（commit `6301cf1`）：序列图 diff 颜色标注 → 用 Note 标注替代 rect rgb()（Mermaid 不支持）

#### 序列图 diff 颜色标注
- Mermaid `sequenceDiagram` 不支持 `style` 注解（classDiagram 支持）
- `rect rgb()` / `rect rgba()` 导致渲染失败或 NaN 坐标
- **最终方案**：`Note right of Participant: 🟢ADDED` 标注变更类型
- 消息前缀：`[+]` 新增、`[~]` 修改、`[-]` 删除

#### Cron 监控问题（反复出现）
- **根因**：prompt 里写 NO_REPLY → 系统吞掉 → 用户看不到通知
- **次要原因**：进程结束后 exec session 被清理，cron 检测不到已完成进程
- **修复尝试**：改 prompt 不写 NO_REPLY，始终报告状态
- **状态**：手动触发测试时 cron 报错（`⚠️ ⏰ Cron failed`），**尚未验证成功**
