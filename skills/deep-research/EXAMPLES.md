# Deep Research 使用示例

## 示例 1: 快速了解 (Quick + 1 轮)

**输入**:
```
研究主题：什么是 MoE (Mixture of Experts)
搜索轮次：1
思考深度：quick
```

**执行**:
1. 分解问题 → "MoE 是什么？核心机制？代表模型？"
2. 搜索 1 次（英文关键词 "Mixture of Experts LLM architecture"）
3. 抓取 top 3 页面
4. 输出 300-500 字快速总结

**预期输出结构**:
```markdown
# MoE (Mixture of Experts) 快速总结

## 一句话定义
MoE 用多个"专家"子网络 + 门控路由器替代单一密集网络，实现更大模型、更低计算成本。

## 核心要点
- 专家网络：多个 FFN 子网络，各自擅长不同输入
- 门控路由：为每个 token 动态选择激活哪些专家（通常 top-2）
- 稀疏激活：推理时只运行部分专家

## 关键数据
- DeepSeek-V3：671B 总参数 / 37B 激活参数
- 推理成本可降至 1/10（NVIDIA Blackwell）

## 核心挑战
- 负载均衡：路由器可能倾向少数专家
- 显存开销：所有专家参数需常驻内存

## 来源
1. [DeepSeek-V3 Technical Report](https://arxiv.org/abs/2412.19437)
2. [Mistral 8x7B Paper](https://arxiv.org/abs/2402.02603)
```

## 优化技巧

1. **聚焦维度** — 减少无关搜索，提升相关性
2. **具体化主题** — "AI 趋势" 太宽 → "2026 AI Agent 框架对比"
3. **不要过度轮次** — 4-5 轮收益递减，3 轮通常足够
4. **中英混合搜索** — 技术话题优先英文，中文话题用 `search_lang="zh-hans"`
